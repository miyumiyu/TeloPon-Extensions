# 🎮 VCI OSC баннеры (VciOscPlugin.py)

📥 **[Скачать VciOscPlugin.py](https://raw.githubusercontent.com/miyumiyu/TeloPon-Extensions/main/plugins/VciOscPlugin.py)**

Этот плагин отправляет баннеры, выводимые ИИ, в реальном времени через протокол **OSC (OpenSound Control)** в **VCI VirtualCast**.
Позволяет отображать баннеры в виртуальном пространстве VirtualCast.

---

## 🛠️ Подготовка

### Необходимые компоненты

1. **VirtualCast** (версия для ПК) установлен и запущен
2. В настройках VirtualCast включена **функция приёма OSC** (порт: 19100)
3. **VCI** для отображения баннеров размещён в VirtualCast

### Настройка OSC в VirtualCast

1. На титульном экране VirtualCast → откройте настройки **«VCI»**
2. Установите **«Функция приёма OSC»** в значение **«Включено»** или **«Только для создателей»**
3. Убедитесь, что порт приёма — **19100** (по умолчанию)

### Скрипт приёма на стороне VCI (Lua)

Настройте следующий Lua-скрипт в VCI для отображения баннеров:

```lua
-- Приём баннеров от TeloPon и их отображение
vci.osc.RegisterMethod("/telopon/telop/text", function(sender, name, args)
    local text = args[1]
    vci.assets.SetText("TextBoard", text)
end, {ExportOscType.String})

-- Приём бейджа (тега эмоции)
vci.osc.RegisterMethod("/telopon/telop/text/badge", function(sender, name, args)
    local badge = args[1]
    -- Эффекты в зависимости от бейджа и т.д.
end, {ExportOscType.String})

-- Приём типа окна
vci.osc.RegisterMethod("/telopon/telop/text/window", function(sender, name, args)
    local window = args[1]
    -- Переключение отображения в зависимости от типа окна и т.д.
end, {ExportOscType.String})
```

> ⚠️ OSC-адрес по умолчанию — `/telopon/telop/text`. Убедитесь, что он совпадает с настройками в TeloPon.

## ⚙️ Настройка и использование в TeloPon

### 1. Откройте панель управления

На главном экране TeloPon в панели «Расширения (плагины)» справа нажмите кнопку **«Панель управления»** у пункта **«VCI OSC баннеры»**.

<img width="400" alt="Экран настроек VCI OSC баннеров" src="../images/VciOscPlugin.png" />

### 2. Включите отправку OSC

Установите флажок **«Отправка OSC ВКЛ»** — при каждом появлении баннера OSC-сообщение будет автоматически отправляться в VCI.

### 3. Настройки подключения

| Настройка | Значение по умолчанию | Описание |
|---|---|---|
| **IP-адрес назначения** | `127.0.0.1` | IP-адрес ПК, на котором работает VirtualCast (если тот же ПК — оставьте как есть) |
| **Порт назначения** | `19100` | Порт приёма OSC в VirtualCast |
| **OSC-адрес** | `/telopon/telop/text` | Должен совпадать с `RegisterMethod` на стороне VCI |

### 4. Тестовая отправка

Нажмите кнопку **«Тестовая отправка»** — будет отправлено тестовое сообщение. Если текст отображается на стороне VCI, подключение успешно.

### 5. Настройки содержимого отправки

| Настройка | По умолчанию | Описание |
|---|---|---|
| **📌 Отправлять также TOPIC** | ВКЛ | При включении отправляется в формате «TOPIC \| MAIN». При отключении — только MAIN |
| **🏷️ Отправлять также бейдж** | ВЫКЛ | При включении бейдж (тег эмоции) отправляется отдельно по адресу `/address/badge` |

### 6. Закрыть

Нажмите кнопку **«Закрыть»** или кнопку **×**, чтобы закрыть панель настроек. Настройки автоматически сохраняются в `plugins/vci_osc.json`.

## 📡 Отправляемые OSC-сообщения

| OSC-адрес | Тип данных | Содержание | Пример |
|---|---|---|---|
| `/telopon/telop/text` | String | Текст баннера (может включать TOPIC) | `Начало трансляции \| Добро пожаловать!` |
| `/telopon/telop/text/badge` | String | Название бейджа (опционально) | `удивление` |
| `/telopon/telop/text/window` | String | Тип окна (отправляется всегда) | `window-simple` |

> 💡 HTML-теги в тексте баннера (`<b1>`, `<b2>`, теги маджонг-плиток и т.д.) автоматически удаляются, и текст отправляется в виде простого текста.

## ❓ Устранение неполадок

### В. При тестовой отправке появляется «❌ Отправка не удалась»
- Проверьте правильность IP-адреса и порта назначения
- Убедитесь, что брандмауэр не блокирует UDP-порт 19100

### В. Текст не отображается на стороне VCI
- Убедитесь, что функция приёма OSC в VirtualCast включена
- Убедитесь, что OSC-адрес в Lua-скрипте VCI совпадает с настройками TeloPon
- Убедитесь, что VCI размещён в VirtualCast

### В. Хочу отправлять на VirtualCast на другом ПК
- Измените IP-адрес назначения на IP-адрес ПК, на котором работает VirtualCast
- Разрешите порт 19100 (UDP) в брандмауэре принимающего ПК
